# ТЖМоделированиеРасш — расширение для МоделированиеТЖ.epf

Расширение конфигурации 1С с одним общим модулем `ВоркерыТЖ`, содержащим экспортные методы-воркеры для запуска через `ФоновыеЗадания.Выполнить`.

## Зачем нужно

Внешняя обработка не может запустить собственный метод во `ФоновыеЗадания.Выполнить()` — платформа требует **общий модуль конфигурации** или **общий модуль активного расширения**. Без этого расширения `МоделированиеТЖ.epf` переключается на fallback: запуск параллельных сессий через `1cv8c.exe /Execute /C"параметр" + WScript.Shell.Run`. Это работает, но:

- Старт каждого воркера занимает 3-10 сек (vs ~100-300 мс для фонового задания)
- Каждый воркер забирает клиентскую лицензию 1С
- Открываются скрытые окна 1С Предприятия

С расширением — фоновые задания идут в `rphost` кластера, не открывают окон, не жрут клиентские лицензии, синхронизируются за миллисекунды.

## Состав

```
src\
├── Configuration.xml              ← манифест расширения (AddOn, режим совместимости 8.3.6)
├── Languages\Русский.xml          ← язык расширения
└── CommonModules\
    ├── ВоркерыТЖ.xml              ← метаданные общего модуля
    └── ВоркерыТЖ\Ext\Module.bsl   ← код экспортных методов
install.ps1                        ← установка в базу (DESIGNER /LoadConfigFromFiles + /UpdateDBCfg)
uninstall.ps1                      ← удаление из базы
common-module.json                 ← исходник для повторной генерации через meta-compile
```

## Экспортные методы общего модуля `ВоркерыТЖ`

| Метод | Параметры (массив) | Что делает |
|-------|---------|-----------|
| `БлокировкаHolder` | `[ИмяРегистра, ВремяСек]` | Берёт исключительную блокировку и удерживает ВремяСек, потом откатывает |
| `БлокировкаWaiter` | `[ИмяРегистра]` | Пытается взять ту же блокировку → пишется TLOCK в ТЖ |
| `ДедлокЭскалация` | `[ИмяРегистра, ПаузаСек]` | S → пауза → X на одном ресурсе (тип 1) |
| `ДедлокРазныйПорядок` | `["Рег1,Рег2", ПаузаСек]` | X на Рег1 → пауза → X на Рег2 (тип 2) |
| `ДедлокОдинРесурс` | `["Основной,Вспомог", ПаузаСек]` | S на вспомог → пауза → X на основном (тип 3) |

Все методы транзакционные, откатывают свои блокировки — в базе ничего не сохраняется.

## Установка в базу Test1CProf

**Закрой Конфигуратор** на Test1CProf если он открыт. Затем:

```powershell
cd D:\1C-Optimyzer\ТЖМоделированиеРасш
.\install.ps1
```

По умолчанию подключение: `localhost:2541\Test1CProf`, пользователь `BVS`, пароль пустой. Если параметры отличаются:

```powershell
.\install.ps1 -Server 'srv01:1541' -Ref 'MyBase' -User 'Admin' -Password 'secret'
```

После установки в Конфигураторе: **Конфигурация → Расширения** — должен появиться `ТЖМоделированиеРасш`.

## Удаление

```powershell
cd D:\1C-Optimyzer\ТЖМоделированиеРасш
.\uninstall.ps1
```

## Пересборка после изменения BSL-кода

Если изменил `src\CommonModules\ВоркерыТЖ\Ext\Module.bsl`:

```powershell
cd D:\1C-Optimyzer\ТЖМоделированиеРасш
powershell -NoProfile -File D:\1C-Optimyzer\1c-optimyzer\.claude\skills\cfe-validate\scripts\cfe-validate.ps1 -ExtensionPath src
.\install.ps1  # перезальёт исходники в базу
```

## Безопасность

- Расширение **AddOn** (дополнение) — не патчит существующие объекты типовой конфигурации, добавляет только свой общий модуль
- Свойство `Privileged = true` у модуля — позволяет работать с блокировками без проверки прав ролей
- Никаких объектов данных не создаётся, никаких обработчиков событий не подписывается
- При удалении расширения база возвращается к исходному состоянию без следов
