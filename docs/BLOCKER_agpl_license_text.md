# Блокер: не могу получить точный текст AGPL-3.0 из сети

**Контекст:** задача `Оптимайзер_смена_лицензии.md` — срочно сменить лицензию на AGPL-3.0 до выхода
статьи на Инфостарте. Требует точного, неизменённого текста лицензии в файле `LICENSE`
(источник — https://www.gnu.org/licenses/agpl-3.0.txt).

**Проблема:** в этой рабочей среде нет сетевого доступа к внешним источникам:
- `curl https://www.gnu.org/licenses/agpl-3.0.txt` → `ECONNREFUSED`.
- `WebFetch` пропускает контент через суммаризирующую модель и не отдаёт файл дословно —
  для юридического текста это неприемлемо (FSF прямо запрещает изменять текст лицензии).

**Что уже сделано (не блокировано):**
- Проверено текущее состояние: `LICENSE` в корне отсутствует; `frontend/src-tauri/Cargo.toml` и
  `backend/pyproject.toml` сейчас помечены `"Proprietary"` (не MIT, как предполагалось в задаче).
- `frontend/package.json`, `cabinet/package.json` — поля `license` нет вовсе.

**Варианты продолжить (нужно решение владельца/архитектора):**
1. Записать текст AGPL-3.0 по памяти модели (документ стабильный и хорошо известный) — с последующей
   ручной сверкой владельцем против gnu.org/licenses/agpl-3.0.txt перед публикацией на GitHub.
2. Оставить в `LICENSE` шапку copyright + пометку `[ВСТАВИТЬ ПОЛНЫЙ ТЕКСТ AGPL-3.0 С gnu.org]`,
   владелец сам вставит текст вручную (скачает у себя, где есть интернет).
3. Подождать, пока в среде появится сетевой доступ, и повторить попытку.
4. Владелец может прислать текст лицензии файлом/вставкой в чат — я вставлю как есть.

**Остальная часть задачи** (README, манифесты, SPDX-заголовки, коммит) не зависит от этого блокера
и может быть выполнена сразу после того, как определимся с текстом LICENSE.
