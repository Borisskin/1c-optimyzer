# CCH Feature Parity Reference — 1C-Optimyzer ↔ ЦУП

> **Назначение:** Карта соответствия между функционалом 1C-Optimyzer и Центром Управления Производительностью (ЦУП) — главным конкурентом в категории мониторинга производительности 1С. Это **competitive reference** для маркетинговых материалов, статей на Инфостарте/Хабре, и для решений по приоритетам спринтов.

## Принципы документа

1. Не сравниваем по «техническому стеку» (это не интересно покупателю). Сравниваем по **тому что пользователь может сделать**.
2. Явно указываем где у нас **преимущества**, где **паритет**, где **отставание**.
3. Каждая позиция привязана к конкретному пункту методики ЦУП — это даёт точную ссылку на ИТС для статей.
4. Документ — **живой**. Обновляется после каждого закрытого спринта.

## Раздел 2.11 ЦУП — Расследование инцидентов производительности

### 2.11.2 Пять симптомов производительности (landing screen)

ЦУП на главном экране показывает 5 симптомов:
1. Долгие операции — есть медленные бизнес-операции
2. Длительные транзакции — есть транзакции > порога
3. Конкуренция за ресурсы — высокое количество ожиданий блокировок
4. Взаимоблокировки — TDEADLOCK events
5. Исключения — растёт количество EXCP

| Симптом | ЦУП | У нас | Статус |
|---|---|---|---|
| Долгие операции | ✅ landing card | ✅ Sprint 3 Top Business Operations | 🟢 паритет |

| Конкуренция за ресурсы | ✅ landing card | ✅ Sprint 2 Locks Timeline | 🟢 паритет |

### 2.12.3 Взаимоблокировки (TDEADLOCK) — типы

ЦУП различает три классических типа deadlock'ов:

| Тип | Описание | ЦУП | У нас | Real-data validation |
|---|---|---|---|---|
| Тип 1 — эскалация блокировки в одной транзакции | Сессия захватила S-блокировку, потом пытается повысить до X | ✅ rule | ✅ Sprint 3 rule `deadlock_lock_escalation` | ✅ Sprint 3.5 (81 TDEADLOCK через симулятор) |
| Тип 2 — захват ресурсов в разном порядке | A блокирует X→Y, B блокирует Y→X | ✅ rule | ✅ Sprint 3 rule `deadlock_different_order` | ✅ Sprint 3.5 |
| Тип 3 — конкуренция двух процессов на одном ресурсе | Оба пытаются взять X-блокировку на одном ресурсе после S | ✅ rule | ✅ Sprint 3 rule `deadlock_single_resource` (fallback) | ✅ Sprint 3.5 |

**Статус раздела 2.12.3:** 🟢 **паритет с ЦУП** + **наше преимущество — AI-объяснение** на каждом классифицированном дедлоке.

Это **сильнейшая позиция в маркетинге**: «Раздел 13 курса 1С:Эксперт реализован полностью, и для каждого дедлока AI объясняет на русском что произошло и что делать — у ЦУПа этой возможности нет».

## Что есть у нас, чего НЕТ у ЦУП — наши конкурентные преимущества

Это самый важный раздел документа для маркетинга.

### Преимущество 1 — AI-объяснения проблем на русском

**ЦУП:** показывает данные, требует от пользователя интерпретации (а интерпретация — это навык 1С:Эксперта).

**Мы:** показываем данные **+ AI-объяснение** что значат данные и что делать. Не требует знаний методики 1С:Эксперт от пользователя.

**Источник преимущества:** прямой доступ к Claude API через грузинский биллинг Сергея — без посредников и без задержек. Российские конкуренты этого делать не могут (санкции, нет доступа к топ-моделям).

**Где видно:** Sprint 3 Anatomy views + ExplainerCard на каждом view.

### Преимущество 2 — AI-rewriter запросов

**ЦУП:** показывает что запрос плохой, рекомендует переписать.

**Мы:** показываем что запрос плохой **+ переписываем его через AI** + side-by-side diff + объяснение каждого изменения.

**Где видно:** Sprint 4 Query Analyzer.

### Преимущество 3 — Standalone desktop, без сервера

**ЦУП:** требует развёртывания серверной части + клиентов + интеграции с ТЖ-сборщиком.

**Мы:** `start.bat` → drag&drop архива → анализ.

**Где видно:** Sprint 0 (initial standalone tool).

### Преимущество 4 — Цена

**ЦУП:** часть КИП (~30 000 ₽/мес для пакета).

**Мы:** целевая цена 5-10 тыс ₽/мес (single product).

**Когда работает:** на ранних этапах продаж, в малых-средних компаниях где КИП избыточен.

### Преимущество 5 — Семантическая валидация запросов против конкретной конфигурации 🆕

**ЦУП:** не делает.

**Мы:** Sprint 5 — пользователь подключает XML выгрузку своей конфигурации, и наш Query Analyzer проверяет что:
- Все объекты в запросе **реально существуют** в этой конфигурации
- Указанные реквизиты есть у соответствующих объектов
- Измерения виртуальных таблиц соответствуют структуре регистра
- Типы в ВЫРАЗИТЬ корректны

Это **уникальная фича в категории**. Сейчас в РФ-сегменте инструментов анализа 1С такого продукта нет.

**Источник преимущества:** наша архитектура backend позволяет загружать XML выгрузку и индексировать её. У конкурентов либо нет такой возможности (они работают только с ТЖ), либо они используют закрытые форматы вроде КИП.

**Где видно:** Sprint 5 ✓ (v0.5.0-internal). Подключение через **Анализ запроса → chip «Конфигурация не подключена» → «Указать папку выгрузки…»**. 8 semantic rules срабатывают автоматически после подключения. Документ для пользователя: `docs/CONNECTING_CONFIGURATION.md`.

### Преимущество 6 — AI-генератор обработок 1С (Sprint 8-9)

**ЦУП:** показывает рекомендации, но не генерирует код.

**Мы:** через MCP BSL Atlas + Claude генерируем **готовые обработки 1С** под конкретную базу пользователя. Не «вот общая рекомендация», а «вот готовый .epf файл который решает эту проблему именно у тебя».

**Где будет:** Sprint 8-9.

**Источник преимущества:** 60+ PowerShell-скиллов для работы с конфигурацией 1С (уже собраны в Sprint 3.5 как побочный продукт), плюс наша инфраструктура AI-доступа.

## Где у ЦУПа есть значимое преимущество — наши слабости

Честно фиксирую gaps чтобы не self-deceive в маркетинге.

### Слабость 1 — Нет real-time мониторинга

**ЦУП:** continuous monitoring, оповещения о проблемах в реальном времени.

**Мы:** только offline-анализ архивов.

**Решается:** Sprint 7 (мониторинг + оповещения с file watcher над папкой ТЖ).

### Слабость 2 — Зрелость и доверие

**ЦУП:** 10+ лет на рынке, использовался крупными компаниями, есть кейсы.

**Мы:** в активной разработке, нет публичных кейсов.

**Решается:** временем + статьями на Инфостарте после Sprint 5.

### Слабость 3 — Wizard настройки logcfg.xml

**ЦУП:** есть встроенный wizard.

**Мы:** инструкция в репо, юзер сам редактирует XML.

**Решается:** Sprint 7 — добавить мастер настройки ТЖ (несложно технически).

### Слабость 4 — Анализ памяти и cache

**ЦУП:** анализирует MEM/LEAKS events, показывает утечки.

**Мы:** не покрываем эти типы событий пока.

**Решается:** Sprint 7-8 — расширение парсера + dedicated view.

### Слабость 5 — Интеграция с КИП экосистемой

**ЦУП:** работает в связке с другими инструментами фирмы 1С (тест-центр, CCTC и др).

**Мы:** standalone, не интегрируется с КИП.

**Решается:** не решается. Это сознательная позиция — мы **не пытаемся** заменить КИП в крупных компаниях, мы для тех у кого КИП нет или избыточен.

## Стратегия конкурентного позиционирования

### Что мы НЕ говорим в маркетинге

- ❌ «Мы заменяем ЦУП». Это раздражает evangelist'ов 1С и преувеличивает нашу зрелость.
- ❌ «Мы лучше чем ЦУП». ЦУП лучше в continuous monitoring + зрелости.
- ❌ «ЦУП устарел». Это неправда и ставит нас в антагонистическую позицию против фирмы 1С.

### Что мы говорим в маркетинге

- ✅ «1C-Optimyzer — это младший брат ЦУПа: для тех у кого нет КИП-лицензии или кому не нужен real-time мониторинг».
- ✅ «Мы покрываем 65% программы курса 1С:Эксперт + добавляем AI-объяснения которых у ЦУПа нет».
- ✅ «Мы делаем то что ЦУП делает с архивами ТЖ — offline-анализ — и делаем это в одном desktop-приложении без серверной части».
- ✅ «Стоимость в 3-6 раз ниже».
- ✅ «AI-объяснения на русском языке — для middle-программистов 1С без сертификации».
- ✅ **(после Sprint 5):** «Единственный продукт в категории, который проверяет запросы против реальной структуры вашей конфигурации».

### Текст для статьи на Инфостарте (черновик, post-Sprint 4)

> **«Альтернатива ЦУП для анализа архивов ТЖ — без КИП-лицензии и без сервера»**
>

Плюс — у нас есть Query Analyzer с AI-rewriter запросов, чего нет ни в ЦУП, ни в других инструментах.

## ADR-029: ЦУП как canonical competitive reference

**Решение:** ЦУП от фирмы 1С (как реализация методики 1С:Эксперт) принимается как **главный конкурентный референс** для всех маркетинговых материалов 1C-Optimyzer.

**Правила:**

1. Каждый маркетинговый материал (статья, лендинг, презентация) явно использует таблицу из этого документа для positioning vs ЦУП.
2. Этот документ обновляется после каждого закрытого спринта в части «Покрытие методики ЦУП по разделам».
3. Текстовые формулировки «1C-Optimyzer — младший брат ЦУПа», «альтернатива для тех у кого нет КИП», «AI-объяснения которых у ЦУПа нет», «единственный продукт с семантической валидацией против конкретной конфигурации» — **canonical phrases** для маркетинга. Их не меняем без явного решения.

**Принято:** Сергей (owner) + Claude Opus 4.7 (architect), 2026-05-20.
