Настройка технологического журнала

Чтобы Optimyzer мог что-то проанализировать — в 1С должен писаться ТЖ. Здесь — рабочие конфиги для типовых сценариев.

Где лежит logcfg.xml

  • Windows: C:\Program Files\1cv8\conf\logcfg.xml или %APPDATA%\1C\1cv8\logcfg.xml
  • Linux: /opt/1cv8/conf/logcfg.xml или ~/.1cv8/1C/1cv8/logcfg.xml

После изменения файла перезапустите сервер 1С (или просто закройте/откройте клиент в файловом режиме). Новый файл подхватывается автоматически.

Базовый конфиг для расследования проблем

Пишем медленные запросы, дедлоки, исключения. Сразу до 50 МБ ротации:

<?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://v8.1c.ru/v8/tech-log">
  <log location="C:\1c-tj-logs" history="48">
    <event>
      <eq property="name" value="DBMSSQL"/>
      <ge property="duration" value="1000000"/>  <!-- >= 1 секунда -->
    </event>
    <event>
      <eq property="name" value="EXCP"/>
    </event>
    <event>
      <eq property="name" value="TDEADLOCK"/>
    </event>
    <event>
      <eq property="name" value="TLOCK"/>
      <ge property="WaitConnections" value="1"/>
    </event>
    <property name="all"/>
  </log>
</config>

Сценарий: разбор медленных операций

Шире: пишем CALL, SCALL, DBMSSQL, EXCP, и detailed контексты:

<?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://v8.1c.ru/v8/tech-log">
  <log location="C:\1c-tj-logs" history="24">
    <event>
      <eq property="name" value="CALL"/>
      <ge property="duration" value="3000000"/>
    </event>
    <event>
      <eq property="name" value="SCALL"/>
      <ge property="duration" value="3000000"/>
    </event>
    <event>
      <eq property="name" value="DBMSSQL"/>
      <ge property="duration" value="500000"/>
    </event>
    <event>
      <eq property="name" value="EXCP"/>
    </event>
    <event>
      <eq property="name" value="EXCPCNTX"/>
    </event>
    <property name="all"/>
  </log>
</config>

Сценарий: охота за дедлоками

Минималистично — только блокировки:

<?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://v8.1c.ru/v8/tech-log">
  <log location="C:\1c-tj-locks" history="72">
    <event><eq property="name" value="TDEADLOCK"/></event>
    <event><eq property="name" value="TLOCK"/></event>
    <event><eq property="name" value="TTIMEOUT"/></event>
    <property name="all"/>
  </log>
</config>

Production-конфиг для continuous monitoring

Optimyzer пока работает offline (импорт архивов). Continuous monitoring будет в Sprint 7+ — следите за новостями.

Если уже хочется писать постоянно — конфиг ниже минимизирует объём:

<?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://v8.1c.ru/v8/tech-log">
  <log location="D:\1c-tj-prod" history="168">  <!-- 7 дней -->
    <event>
      <eq property="name" value="DBMSSQL"/>
      <ge property="duration" value="3000000"/>  <!-- 3 секунды -->
    </event>
    <event><eq property="name" value="TDEADLOCK"/></event>
    <event><eq property="name" value="EXCP"/></event>
    <property name="all"/>
  </log>
</config>

Сколько весит ТЖ

  • Базовый конфиг: ~100-500 МБ/сутки на средней нагрузке
  • Полный (всё подряд): несколько ГБ/час — НЕ рекомендуется

Архив за неделю обычно 10-50 ГБ — это нормально для Optimyzer.
