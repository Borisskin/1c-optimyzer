AI Explainer

Главная фишка Optimyzer — AI читает данные конкретной проблемы и объясняет на русском: что произошло, почему, что чинить.

Когда AI вызывается

  • На экране «Анатомия операции» — кнопка «Объяснить через AI»
  • На экране «Анатомия дедлока» — автоматически, или вручную
  • На экране «Медленные запросы» — раскрытие строки

Что передаётся в Claude

В Claude API уходит только агрегированный контекст конкретной проблемы:

  • Тип события (CALL, EXCP, TDEADLOCK)
  • Суммарные метрики: duration, ms, EXCP count
  • Топ-N SQL-запросов (нормализованных, без значений параметров)
  • Stack trace / lock graph для дедлоков

Что НЕ передаётся

  • Содержимое значений параметров запросов
  • Имена баз 1С, имена пользователей
  • Полное содержимое контекстов (Context поля)
  • Содержимое EXCP messages с конкретными данными
  • Любые данные из других сессий / архивов

Кеш ответов

AI-ответы сохраняются локально в data/explainer_cache.db. Если открыть ту же анатомию снова — ответ покажется мгновенно из кеша, без обращения к Claude и без списания квоты/кредитов.

Лимиты

  • Free: 5 AI-объяснений в месяц. После — paywall с предложением Pro/Credits.
  • Pro: безлимит до soft cap 1000/мес. После — warning в личном кабинете, но операции продолжают работать.
  • Credits: 1 credit = 1 операция. Кеш всё равно работает — повторный запрос бесплатно.

Качество ответов

Используется Claude 4.7 (Sonnet) с системным промптом «эксперт по 1С». Ответы:

  • На русском языке
  • Со ссылками на конкретные строки данных
  • С предложениями «что сделать» (изменить запрос, поднять памяти кеша конфигурации, добавить индекс и т.п.)

AI не идеален и может галлюцинировать. Всегда сверяйтесь с фактическими данными в Анатомии — числа всегда корректны (их собирает не AI, а наш парсер).
