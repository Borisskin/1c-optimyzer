# Неявное декартово произведение через запятую

## Что не так

`ИЗ Таблица1 КАК А, Таблица2 КАК Б` — это **CROSS JOIN** (декартово произведение). Если таблицы по миллиону строк — получишь 10¹² комбинаций до того как WHERE их отфильтрует. Catastrophic для производительности.

## Почему это плохо

Декартово произведение материализуется СУБД **до** применения `ГДЕ`. План запроса будет: `Cartesian Product → Filter`. Это O(N*M) memory + CPU. Часто такой запрос «вешает» сервер или вылетает по памяти.

Кроме того, синтаксис через запятую — устаревший. Современные оптимизаторы хуже понимают намерение разработчика.

## Как переписать

Всегда используй явный `СОЕДИНЕНИЕ`:

```sdbl
-- Плохо:
ВЫБРАТЬ ...
ИЗ Документ.Реализация КАК Реал, Справочник.Контрагенты КАК Контр
ГДЕ Реал.Контрагент = Контр.Ссылка

-- Хорошо:
ВЫБРАТЬ ...
ИЗ Документ.Реализация КАК Реал
    ВНУТРЕННЕЕ СОЕДИНЕНИЕ Справочник.Контрагенты КАК Контр
        ПО Реал.Контрагент = Контр.Ссылка
```
