# Оператор В с подзапросом

## Что не так

`В (ВЫБРАТЬ ...)` в SDBL транслируется в SQL `IN (SELECT ...)`, что в MS SQL и PostgreSQL часто превращается в semi-join с не-оптимальным планом. Кроме того, подзапрос в `В` не имеет индекса и оптимизатор не всегда умеет его материализовать.

## Почему это плохо

- Подзапрос внутри `В` выполняется для каждой строки внешней таблицы (в плохих планах).
- Дубликаты в результате подзапроса искажают cardinality estimation.
- `НЕ В (ВЫБРАТЬ)` ещё хуже — антипаттерн почти всегда.

## Как переписать

### Замена на ВНУТРЕННЕЕ СОЕДИНЕНИЕ

Если хотим оставить только те строки которые есть в обеих таблицах:

```sdbl
-- Плохо:
ВЫБРАТЬ ...
ИЗ Документ.Реализация
ГДЕ Контрагент В (ВЫБРАТЬ Контрагент ИЗ Справочник.НашиКлиенты)

-- Лучше:
ВЫБРАТЬ ...
ИЗ Документ.Реализация КАК Реал
    ВНУТРЕННЕЕ СОЕДИНЕНИЕ Справочник.НашиКлиенты КАК Кл
        ПО Реал.Контрагент = Кл.Ссылка
```

### Если нужно `В` именно из набора значений

Используй временную таблицу с индексом:

```sdbl
ВЫБРАТЬ Ссылка ПОМЕСТИТЬ ВТ_Клиенты ИЗ Справочник.НашиКлиенты
ИНДЕКСИРОВАТЬ ПО Ссылка;

ВЫБРАТЬ ...
ИЗ Документ.Реализация КАК Реал
    ВНУТРЕННЕЕ СОЕДИНЕНИЕ ВТ_Клиенты КАК Кл
        ПО Реал.Контрагент = Кл.Ссылка
```
