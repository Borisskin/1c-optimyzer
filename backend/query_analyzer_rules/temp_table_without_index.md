# Временная таблица без ИНДЕКСИРОВАТЬ ПО

## Что не так

Создание временной таблицы через `ПОМЕСТИТЬ ВТ_X` без явного `ИНДЕКСИРОВАТЬ ПО` означает что СУБД не построит индекс. Когда в следующем запросе ВТ_X используется в `СОЕДИНЕНИЕ` — это **full scan** временной таблицы для каждой строки внешней.

## Почему это плохо

Временные таблицы часто содержат тысячи или миллионы строк. Без индекса по полям соединения каждый JOIN превращается в декартово произведение которое СУБД фильтрует уже после.

## Как исправить

Добавь `ИНДЕКСИРОВАТЬ ПО` по тем полям которые используются в условии соединения:

```sdbl
-- Плохо:
ВЫБРАТЬ
    Номенклатура,
    КоличествоОстаток
ПОМЕСТИТЬ ВТ_Остатки
ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(&Дата)
;

-- Хорошо:
ВЫБРАТЬ
    Номенклатура,
    Склад,
    КоличествоОстаток
ПОМЕСТИТЬ ВТ_Остатки
ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(&Дата)
ИНДЕКСИРОВАТЬ ПО Номенклатура, Склад
;
```

## Когда индекс НЕ нужен

- Если временная таблица одноразовая и используется только в `ИЗ ВТ_X` без JOIN — индекс лишняя работа.
- Если ВТ_X содержит < 100 строк и используется один раз — оптимизатор и так справится.
