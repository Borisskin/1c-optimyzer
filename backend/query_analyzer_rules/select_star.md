# ВЫБРАТЬ * — все колонки

## Что не так

`ВЫБРАТЬ *` (или `SELECT *`) выбирает **все колонки** таблицы. В 1С таблицы часто содержат десятки колонок (включая `Проведён`, `ПометкаУдаления`, `МоментВремени`, реквизиты табличной части, итд). Извлечение всех колонок — это:

- Лишние байты передаваемые из СУБД клиенту.
- Невозможность использовать **покрывающий индекс** (covering index).
- Возможный перебор bloated TOAST в PostgreSQL.

## Почему это плохо

Покрывающий индекс — это индекс который содержит **все** колонки выборки. Если запрос `ВЫБРАТЬ Артикул ИЗ Тов ГДЕ Группа = &Г`, СУБД может ответить на запрос **только из индекса** (`Group, Article`), не трогая основную таблицу. С `ВЫБРАТЬ *` это невозможно — нужны все колонки, придётся читать heap.

## Как переписать

Перечисли только нужные колонки:

```sdbl
-- Плохо:
ВЫБРАТЬ *
ИЗ Справочник.Номенклатура
ГДЕ Группа = &Группа

-- Лучше:
ВЫБРАТЬ
    Ссылка,
    Артикул,
    Наименование
ИЗ Справочник.Номенклатура
ГДЕ Группа = &Группа
```
