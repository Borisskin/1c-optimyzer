# ОБЪЕДИНИТЬ без ВСЕ

## Что не так

`ОБЪЕДИНИТЬ` (без `ВСЕ`) делает **`SELECT DISTINCT`** на объединённом результате — убирает дубли. Это аналогично `SELECT DISTINCT UNION ALL`: СУБД сортирует или хеширует **весь результат** обоих запросов для удаления дублей.

## Почему это плохо

В 90% случаев пользователь хочет просто объединить два набора, дубли невозможны (потому что выборки фильтрованы по разным условиям или ключам). Тогда `ОБЪЕДИНИТЬ ВСЕ` даёт **тот же результат** в разы быстрее.

## Как переписать

```sdbl
-- Если ты уверен что дубли невозможны:
ВЫБРАТЬ ... ИЗ Документ.А
ОБЪЕДИНИТЬ ВСЕ
ВЫБРАТЬ ... ИЗ Документ.Б

-- Только если дубли реально возможны и их нужно убрать — оставляй ОБЪЕДИНИТЬ:
ВЫБРАТЬ ... ИЗ Документ.А ГДЕ Сумма > 0
ОБЪЕДИНИТЬ
ВЫБРАТЬ ... ИЗ Документ.А ГДЕ Контрагент = &К
```
