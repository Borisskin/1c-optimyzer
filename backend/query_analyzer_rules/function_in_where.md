# Функция от поля в условии ГДЕ

## Что не так

`ГДЕ ГОД(Док.Дата) = 2024` или `ГДЕ ВРЕГ(Тов.Артикул) = "A001"` — это **функция от индексированного поля**. СУБД не может использовать индекс если в условии стоит функция от колонки — индекс построен по самим значениям, не по `function(value)`.

## Почему это плохо

- Отключение индекса → table scan.
- Каждая строка таблицы проходит через функцию (CPU overhead).
- При большом объёме таблицы — катастрофическая просадка.

## Как переписать

### Дата → диапазон

```sdbl
-- Плохо:
ГДЕ ГОД(Док.Дата) = 2024

-- Хорошо:
ГДЕ Док.Дата >= ДАТАВРЕМЯ(2024, 1, 1)
  И Док.Дата < ДАТАВРЕМЯ(2025, 1, 1)
```

### Строки → точный регистр

Если регистр важен — приведи `&параметр` к нужному регистру **до** запроса, не саму колонку:

```sdbl
-- Плохо:
ГДЕ ВРЕГ(Тов.Артикул) = ВРЕГ(&Артикул)

-- Лучше: контролируй регистр на клиенте, поле сравнивается напрямую:
ГДЕ Тов.Артикул = &АртикулВВерхнемРегистре  -- параметр уже нормализован
```

### Подстрока → like

`ПОДСТРОКА(Тов.Артикул, 1, 3) = "ABC"` → `Тов.Артикул ПОДОБНО "ABC%"` — `LIKE` с префиксом использует индекс.
