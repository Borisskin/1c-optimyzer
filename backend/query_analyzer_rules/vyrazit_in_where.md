# ВЫРАЗИТЬ в условии ГДЕ

## Что не так

`ВЫРАЗИТЬ(... КАК ...)` в условии `ГДЕ` — антипаттерн. Это преобразование типа (cast), которое **отключает использование индекса** по полю и заставляет СУБД делать table scan.

## Почему это плохо

Когда колонка имеет составной тип (например `Ссылка` в Документ может ссылаться на Контрагенты ИЛИ Сотрудники), `ВЫРАЗИТЬ(Ссылка КАК Справочник.Контрагенты)` помогает 1С скомпилировать корректный SQL. Но если использовать в `ГДЕ`, индекс по полю не работает — каждая строка должна быть «приведена к типу» перед сравнением.

## Как переписать

### Вариант 1: фильтр по типу через ССЫЛКА

```sdbl
-- Плохо:
ГДЕ ВЫРАЗИТЬ(Док.Контрагент КАК Справочник.Контрагенты).Наименование = "ООО Рога"

-- Лучше:
ГДЕ Док.Контрагент ССЫЛКА Справочник.Контрагенты
  И Док.Контрагент.Наименование = "ООО Рога"
```

`ССЫЛКА` — это специальный 1С-оператор, который проверяет тип без отключения индексов.

### Вариант 2: фильтр в подзапросе с уже выраженным типом

```sdbl
ВЫБРАТЬ
    ВЫРАЗИТЬ(Док.Контрагент КАК Справочник.Контрагенты) КАК Контрагент,
    Док.СуммаДокумента
ПОМЕСТИТЬ ВТ_Документы
ИЗ Документ.РеализацияТоваровУслуг КАК Док
ГДЕ Док.Контрагент ССЫЛКА Справочник.Контрагенты
ИНДЕКСИРОВАТЬ ПО Контрагент
;

ВЫБРАТЬ ... ИЗ ВТ_Документы ГДЕ Контрагент.Наименование = "ООО Рога"
```
