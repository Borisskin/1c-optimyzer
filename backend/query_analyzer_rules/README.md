# Native Rules для Query Analyzer (Sprint 4)

Каждый `.md` файл — одно правило с YAML frontmatter и markdown-телом.

## Формат

```markdown
---
id: unique_rule_id                  # snake_case, должен быть уникальным
severity: warning                   # critical | warning | info
category: performance               # performance | correctness | style
patterns:
  - '(?im)REGEX_PATTERN_1'
  - '(?im)REGEX_PATTERN_2'
tags: [tsup-2.13.4, expert-10]
---

## Severity

- `critical` — точно плохо, потеря производительности на порядки
- `warning` — почти всегда плохо, но есть оправданные случаи
- `info` — стоит обратить внимание, иногда оправдано

## Patterns

Regex с флагами `(?i)` (case-insensitive) и `(?m)` (multiline) обязательны
для матчинга русских ключевых слов в любом регистре. Для multi-line паттернов
используй `(?ims)` с `s` (dotall) и заранее ограничивай жадность через `.{0,N}?`.

## Tags

- `tsup-2.13.4` — методика ЦУП «Корпоративный инструментальный пакет», раздел 2.13.4
- `expert-10` — Раздел 10 курса 1С:Эксперт «Запросы которые работают быстро»
- `expert-7` — Раздел 7 курса «Индексы»
- `expert-8` — Раздел 8 курса «Анализ плана запроса»

## Что НЕ делать в правилах

- Не пиши слишком жадные regex — false positives раздражают.
- Не покрывай ВСЕ возможные варианты одним правилом — лучше разбить.
- Не используй `^` / `$` без `(?m)` — иначе работает только на весь текст.
- Не забывай экранировать `(`, `)`, `.`, `+`, `?` если они литералы.
