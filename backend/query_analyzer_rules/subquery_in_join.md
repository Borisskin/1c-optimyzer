# Соединение с подзапросом

## Что не так

`СОЕДИНЕНИЕ (ВЫБРАТЬ ...)` — это inline-подзапрос внутри JOIN. СУБД 1С (особенно MS SQL и PostgreSQL) часто не умеет вынести такой подзапрос в материализованный набор. Результат — повторное выполнение подзапроса для каждой строки внешней таблицы.

## Почему это плохо

Подзапрос внутри соединения не индексирован. Оптимизатор СУБД не имеет статистики и плана для этого подзапроса. В большинстве случаев это означает full scan левой таблицы помноженный на full scan подзапроса.

## Как переписать

Вынести подзапрос в **временную таблицу** с явным индексом по полям соединения:

```sdbl
ВЫБРАТЬ
    Док.Контрагент,
    СУММА(Док.СуммаДокумента) КАК СуммаДолга
ПОМЕСТИТЬ ВТ_Долги
ИЗ
    Документ.СчётНаОплату КАК Док
ГДЕ
    Док.Проведён = ИСТИНА
СГРУППИРОВАТЬ ПО Док.Контрагент
ИНДЕКСИРОВАТЬ ПО Контрагент
;

ВЫБРАТЬ ...
ИЗ
    Справочник.Контрагенты КАК Контрагенты
    ВНУТРЕННЕЕ СОЕДИНЕНИЕ ВТ_Долги КАК Долги
        ПО Контрагенты.Ссылка = Долги.Контрагент
```
