# Значение перечисления не существует

## Что не так

В запросе используется `{{object_full_name}}.{{value_name}}`, но значения `{{value_name}}` нет у перечисления `{{object_full_name}}`.

## Почему это плохо

При сравнении с несуществующим значением:

- **Запрос вернёт пустой результат** — что часто маскирует баг (отчёт «работает», но показывает 0 строк всегда).
- **На уровне SDBL платформа выдаст ошибку**, если перечисление есть, а значение нет — запрос не выполнится.
- **Запрос из другой конфигурации** часто содержит значения, которых нет в текущей базе.

## Допустимые значения

{{valid_values}}

## Как исправить

1. Проверьте имя значения — возможно опечатка.
2. Если значение нужно, но его нет — добавьте через Конфигуратор (Перечисление → Значения → Добавить).
3. Если запрос из другой конфигурации — найдите аналог среди допустимых значений по бизнес-смыслу.
