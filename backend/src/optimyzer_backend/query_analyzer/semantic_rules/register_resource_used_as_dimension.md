# В параметре виртуальной таблицы — ресурс вместо измерения

## Что не так

В параметрах виртуальной таблицы `{{object_full_name}}.Остатки(...)` используется фильтр по `{{field_name}}`, но `{{field_name}}` — **ресурс** регистра, а не измерение.

## Почему это плохо

Виртуальные таблицы регистров принимают в параметрах только **измерения** — поля, по которым регистр индексирован. Фильтр по ресурсу:

- **Не имеет смысла как параметр виртуальной таблицы** — ресурсы используются для агрегации (СУММА, КОЛИЧЕСТВО), а не для фильтрации
- В некоторых сборках платформы **тихо игнорируется** — запрос отдаёт неправильный результат, но не падает
- **Замедляет выполнение** — full scan регистра

{{available_dimensions}}

## Как исправить

Перенесите фильтр по ресурсу из параметров виртуальной таблицы в `ГДЕ` снаружи:

```sdbl
ВЫБРАТЬ ...
ИЗ {{object_full_name}}.Остатки(&Дата, ) КАК Т
ГДЕ Т.{{field_name}} > 0
```

Если же нужно агрегировать по этому ресурсу — используйте `СУММА(Т.{{field_name}})` в `ВЫБРАТЬ` и `СГРУППИРОВАТЬ ПО` по измерениям.
