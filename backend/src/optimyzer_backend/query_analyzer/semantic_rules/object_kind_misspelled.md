# Тип объекта написан во множественном числе

## Что не так

В SDBL-запросе тип объекта `{{wrong_kind}}` написан во **множественном числе**. В языке запросов 1С используется **единственное число**: `{{correct_kind}}`.

Правильно: `{{correct_full_name}}`

## Почему это плохо

Запрос **не выполнится** — платформа 1С выдаст ошибку «Неизвестный тип» или «Таблица не найдена». Это очень частая опечатка у программистов, особенно которые недавно начали работать с языком запросов 1С (в коде на BSL действительно используется `Справочники.Контрагенты.НайтиПоНаименованию(...)` через менеджер, но в **запросах** — только `Справочник.Контрагенты`).

## Как исправить

Замените `{{wrong_kind}}.{{object_name}}` на `{{correct_full_name}}`.

## Список правильных типов в SDBL

- `Справочник` (не «Справочники»)
- `Документ` (не «Документы»)
- `Перечисление` (не «Перечисления»)
- `РегистрНакопления` (не «РегистрыНакопления»)
- `РегистрСведений` (не «РегистрыСведений»)
- `РегистрБухгалтерии` (не «РегистрыБухгалтерии»)
- `РегистрРасчета` (не «РегистрыРасчета»)
- `ПланСчетов`, `ПланВидовХарактеристик`, `ПланВидовРасчета`, `ПланОбмена`
- `ЖурналДокументов`, `Константа`, `БизнесПроцесс`, `Задача`, `Последовательность`
