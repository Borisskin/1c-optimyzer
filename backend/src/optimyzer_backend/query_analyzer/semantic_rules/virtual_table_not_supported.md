# Виртуальная таблица недопустима для этого регистра

## Что не так

Запрос обращается к `{{object_full_name}}.{{virtual_table}}`, но у этого объекта **нет такой виртуальной таблицы**.

Допустимые виртуальные таблицы для `{{object_full_name}}`: {{valid_virtual_tables}}.

## Почему это плохо

Виртуальные таблицы зависят от типа регистра:

- **Регистр накопления (Balance)** — `Остатки`, `Обороты`, `ОстаткиИОбороты`
- **Регистр накопления (Turnovers)** — только `Обороты` (остатков нет)
- **Регистр сведений** — `СрезПоследних`, `СрезПервых`
- **Регистр бухгалтерии** — `Остатки`, `Обороты`, `ОстаткиИОбороты`, `ДвиженияССубконто`, `ОборотыДтКт`
- **Регистр расчёта** — `БазаДанных`, `ДанныеГрафика`, `ФактическийПериодДействия`

Запрос с неподдерживаемой виртуальной таблицей **не выполнится** — будет синтаксическая ошибка SDBL.

## Как исправить

1. Проверьте тип регистра — это регистр остатков или оборотов?
2. Замените виртуальную таблицу на допустимую (из списка выше).
3. Если нужны срезы во времени — рассмотрите регистр сведений вместо регистра накопления.
