# Объект не существует в подключённой конфигурации

## Что не так

Запрос ссылается на объект `{{object_full_name}}`, которого нет в подключённой XML-выгрузке конфигурации.

## Почему это плохо

Возможные причины:

1. **Опечатка в имени** — попробуйте уточнить написание.
2. **Запрос скопирован из другой конфигурации.** Например, в УТ есть `РегистрНакопления.ТоварыНаСкладах`, а в БП 3.0 — нет: данные о товарах в БП лежат в плане счетов через `РегистрБухгалтерии.Хозрасчетный`.
3. **Объект был удалён или переименован** — проверьте свежесть XML-выгрузки. Если давно её делали, перевыгрузите из Конфигуратора.
4. **Расширение или дополнение** добавляет объект в runtime, но в XML-выгрузке его нет.

В любом случае такой запрос **не выполнится в вашей базе** — будет ошибка «Объект не найден».

## Как исправить

1. Если есть похожий объект из списка выше — заменить и проверить структуру (реквизиты могут отличаться).
2. Если запрос из другой конфигурации — найти аналог по бизнес-смыслу. Спросите у архитектора или сверьтесь с метаданными вашей базы.
3. Если выгрузка устарела — перевыгрузите из Конфигуратора (Конфигурация → Выгрузить конфигурацию в файлы XML).
