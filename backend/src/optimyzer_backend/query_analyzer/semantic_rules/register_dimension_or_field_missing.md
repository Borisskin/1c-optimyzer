# Параметр виртуальной таблицы — не измерение регистра

## Что не так

В виртуальной таблице регистра `{{object_full_name}}` используется фильтр по `{{field_name}}`, но `{{field_name}}` — **не измерение** этого регистра.

## Почему это плохо

Виртуальные таблицы регистров (`Остатки`, `Обороты`, и т.п.) принимают в параметрах **только измерения** — индексы, по которым регистр оптимизирован. Фильтр по реквизиту или ресурсу в параметрах:

- **Не использует индекс** — приводит к full scan регистра
- **Даёт некорректный результат** — фильтрация после агрегации, а не до
- **Тихо игнорируется платформой** в некоторых сборках 1С (что хуже всего — баг сидит в продакшене)

## Какие измерения есть у этого регистра

{{available_dimensions}}

## Как исправить

1. Если по полю **нужен** фильтр на уровне виртуальной таблицы — сделать его измерением регистра (через Конфигуратор). Это требует обновления конфигурации, но даст огромный выигрыш по производительности.
2. Если поле — реквизит регистра, фильтр перенести в `ГДЕ` снаружи виртуальной таблицы:

```sdbl
ВЫБРАТЬ ... ИЗ РегистрНакопления.Х.Остатки(, ) КАК Т
ГДЕ Т.{{field_name}} = &Значение
```

3. Если поле — ресурс, см. правило `register_resource_used_as_dimension`.
